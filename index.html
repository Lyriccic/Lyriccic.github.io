单例设计模式

  • 设计模式：是一种代码结构，编程风格和思考模式》》》套路

  • 单例的设计模式是指：采取一定的方法保证整个软件系统中只能存在一个对象实例，

    首先要私有化构造器，这是为了防止在类的外面用构造器new出多个对象。然后内部创建类的对象，要私有化和静态化。第三步，提供静态方法方法返回类的对象。

    • ```java
      //饿汉式
      class Bank{
      private Bank(){//私有化构造器

}
private static Bank instance = new Bank();//内部创建私有静态的类的对象
public static Bank getInstance(){//静态公共方法返回类的对象
    return instance;//因为方法是static的，所以对象也必须是static的。方法是static的原因是因为，如果不是static的话就必须创建对象调用，但是只有调用这个方法才能创建对象，陷入死锁
      } 这里bank1和bank2是同一个对象，因为用的是static创建的类的对象，这个都西昂适合类一起加载的。
    • ```java
      //懒汉式
      class Order{
      private Order(){//私有化构造器，防止在外面用new创建对象
    
}
private static Order instance = null;//声明当前类的对象
public static Order getInstance(){
    if(instance == null){instance = new Bank();}//为了防止创建多个对象，需要对对象进行判断，如果是已经创建了的对象那么比不可能为null，这样就不会再创建对象了
    return instance
}
      }

    这两种赋值方式的执行顺序是看哪一个写在前面哪个就先执行

static关键字的使用

  1. 静态的

  2. 用来修饰属性、方法、代码块、内部类：

    • 静态变量（类变量）：static修饰的属性：多个对象共享同一个静态变量，当通过某一个对象修改静态变量的时候，通过另一个对象调用的时候该属性已经发生改变

      由于静态变量随着类加载而加载，早于对象的加载，所以可以通过类名直接调用。

      因为类只会加载一次，所以静态变量在内存只存在一份，存在于方法区的静态域中。

      插一句方法区独立于栈和堆，方法区中存储的时候类的加载信息、静态域、常量池

      实例变量：没有static修饰的属性：每个对象对应一个，修改一个不会影响另一个

    • 静态方法：也是随着类加载而加载，可以通过类直接调用，也可以通过对象调用，和静态属性类似

      静态方法中只能调用静态方法和静态属性（因为他们生命周期相同，可能在调用的时候非静态还没产生，这样就不好了），非静态方法静态非静态都可以调用

书上的注释告诉我自动装箱规范要求boolean、byte/char<=127，介于-128和127之间的short和int被包装到固定的对象中，也就是说上面的代码中a和b是被包装到同一个对象中了，a和b只是这个对象的不同引用，所以==操作符比较两个对象是否相同一定是true。

但是如果a和b被包装到同一个对象中，a和b是同一个对象的两个引用的话，如果a改变b也应该跟着改变，可是实验结果并不是这样

csdn上查出来的：是因为，
一来：Integer里弄了一个缓存，对于在 -128—127 之S间的数值，会直接使用该缓存里的对象 ；
所以也就是说 在我们比较 integer = 3 的时候，其实拿取的是同一个对象，所以会返回true。
二来：因为30000超出了 -128～127这个区间，那么系统会重新生成新的对象，所以比较的时候就会拿取不同的对象做比较，因此就会返回false。

有关super 和 this

  1. super调用父类中的方法，只会调用离自己最近的那个父类中的方法
  2. super调用父类构造器可以根据参数的不同调用父类中的不同构造器，前提是父类中有这样的构造器
  3. this 调用本类重载的其他构造器（有可能会在构造器中用this调用的，这里的其他是为了排除本身所在的构造器）
  4. this用来调用本类中的方法和属性
  5. 如果要在构造器中使用this或者super调用构造器，那他们必须写在类的首行，如果既不写this也不写super其实默认写了super空参数
    因为加载子类之前会把他的所有超类先加载了。总之无论如何，都会先调用父类构造器

有关多态

包括方法的重载和覆写还有对象的多态性

对象的多态性

子类的对象可以赋值给父类的变量 ，这样的变量依然是一个声明为超类的变量，他不能直接调用子类中的方法或者字段，如果一定要调用需要强制类型转换向下转型。但是不能将超类的引用赋值给子类的变量 。实际上就是向上转型和向下转型，其中子类对象赋值给父类变量是向上转型，程序会自动完成。向下转型是指把已经赋上了子类对象的父类变量用强制类型转换强制转换为子类变量并且赋值给子类变量

Father father = new Son() //向上转型

Son son = (Son) father //向下转型
为了避免向下转型异常，可以使用instanceof操作符先进行条件判断。

父类引用指向子类对象时，通过子类对象只能调用父类和子类中共有的方法，而且实际是子类重写过的那个，无法调用子类特有的方法！（因为这个引用是父类，编译的时候看的是左边，运行看右边）
就是编译的时候看左边类型，检查类型中是否存在需要调用的字段或者方法，运行的时候看右边具体是什么对象。** 但是对象的多态性只适用于方法不适用于属性 **，也就是说如果通过上述调用调用了一个属性，那么得到的将是父类中的那个。

  • 另外多态性是一个运行时行为 *
    动态绑定是在运行时才确定》》》重载
    静态绑定是在编译时就确定》》》重写

hashCode方法

散列码（hash code）是对象导出的一个整型值，无规律，如果x和y是两个不同的对象，那么x.hashCode()和y.hashCode()不会相同

string的散列码是由内容导出的，因为string类中重写了equals方法和hashCode方法，因此两个字符串内容相同的话他们的散列码是相同的。但是一般的其他类型是散列码是由对象的存储地址导出的
